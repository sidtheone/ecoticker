- **Observation:** The plan says "flat rows, no cards" for dimensions and "left-border accent colored by severity" — replicating HeroSection's `borderLeft: 4px solid ${colors.border}` pattern. But do individual dimension rows need to exist at all? The topic detail page already has the overall score (85, giant text), the insight lede (overall summary), and the SeverityGauge. The three dimensions (Ecological 40%, Health 35%, Economic 25%) are weighted sub-scores that produce the overall score the user already sees. On the landing page, the equivalent data (per-topic scores) earns its place because each row is a *different navigable entity* — you click through. Dimension rows are terminal: there's nothing behind them. They're informational dead-ends. The reasoning text (e.g., "Unprecedented ice loss disrupting polar ecosystems") is the only unique content, but the plan says "tap to expand reasoning" on mobile — burying the only content that justifies the dimension rows' existence behind a tap. VALUES.md says "Subtract until it breaks, then add one back" and "If it doesn't change how you think, it doesn't belong." Ask: if you deleted the entire dimension breakdown section and instead appended the three reasoning sentences to the insight lede as a single paragraph, what would the user lose? They'd lose the individual sub-scores (80, 70, 75) which are already synthesized into the overall 85 they see in giant text. They'd lose the severity levels per dimension (SEVERE, SIGNIFICANT) which are already represented by the overall urgency badge. They'd keep the reasoning — the only part that adds insight. The redesign plan is spending effort restyling an element whose existence hasn't survived the values test.

The flat-row redesign makes the dimensions *look* like the landing page's topic list, creating a visual false equivalence — rows that look navigable but aren't. Users will instinctively try to tap/click dimension rows expecting drill-down, hitting a dead end. The "tap to expand reasoning" interaction adds a click to reach the only valuable content. If the dimension section doesn't survive the Existence Question, all the test rewrites and CSS changes are waste. Recommendation: before implementing the flat-row dimension redesign, run the delete probe — remove the dimension section entirely, fold the reasoning sentences into the insight lede, and evaluate whether the page loses anything that changes how the user thinks. If it does, the section survives and the redesign proceeds. If it doesn't, you just saved ~80 lines of page.tsx and ~36 test assertions.
