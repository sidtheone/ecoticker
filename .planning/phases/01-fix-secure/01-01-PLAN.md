  <action>
    **auth.ts changes:**
    Replace `return apiKey === adminKey` (line 19) with timing-safe comparison:
    ```typescript
    import crypto from 'crypto';
    ```
    In the function body, after the existing `!adminKey` check, add a check for `!apiKey` returning false. Then:
    ```typescript
    const apiKeyBuf = Buffer.from(apiKey);
    const adminKeyBuf = Buffer.from(adminKey);
    if (apiKeyBuf.length !== adminKeyBuf.length) return false;
    return crypto.timingSafeEqual(apiKeyBuf, adminKeyBuf);
    ```
    This handles the edge case where timingSafeEqual throws on unequal buffer lengths.

    **cron/batch/route.ts changes:**
    In the GET handler, when constructing the NextRequest for batchPOST (line 49-51), inject the admin key header:
    ```typescript
    const batchRequest = new NextRequest(new URL("/api/batch", request.url), {
      method: "POST",
      headers: { "x-api-key": process.env.ADMIN_API_KEY || "" },
    });
    ```
    Apply the same fix to the seedPOST request (line 56-58) and to both requests in the POST handler (lines 117-119 and 123-125).
